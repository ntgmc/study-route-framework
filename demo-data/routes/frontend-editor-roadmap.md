# 前端编辑器能力路线

## 路线目标

在 4 周内完成一个可持续迭代的 Markdown 编辑体验，覆盖编辑、预览、保存、草稿、搜索和本地偏好迁移。

## 适用对象

- 已掌握 React 基础，希望提升复杂交互实现能力。
- 需要练习 CodeMirror、IndexedDB、Service Worker 和浏览器自动化测试。

## 阶段路线

| 阶段 | 周期 | 主题 | 关键产出 | 验收标准 |
| --- | --- | --- | --- | --- |
| 1 | 第 1 周 | 编辑器基础 | Markdown 工具栏、快捷键、实时预览 | 能完成常见 Markdown 编辑并即时预览 |
| 2 | 第 2 周 | 本地可靠性 | 自动保存、草稿版本、离线缓存 | 断网或刷新后能恢复最近编辑内容 |
| 3 | 第 3 周 | 内容管理 | 文件树、搜索、标签、收藏、置顶 | 能在 50+ 文件中快速定位内容 |
| 4 | 第 4 周 | 质量保障 | E2E 测试、性能拆分、发布文档 | PR 自动完成检查，构建无关键警告 |

## 每周任务

- 第 1 周：补齐编辑器工具栏、快捷键、分屏预览和滚动同步。
- 第 2 周：把草稿和版本写入 IndexedDB，加入保存状态提示。
- 第 3 周：实现标签、收藏、置顶和元数据导入导出。
- 第 4 周：加入 CI、E2E 用例和变更日志。

## 复盘问题

- 哪些状态必须持久化，哪些只应该存在内存里？
- 编辑区和预览区滚动同步使用行号映射还是滚动比例更稳定？
- 哪些交互需要 toast，哪些只需要状态栏反馈？
