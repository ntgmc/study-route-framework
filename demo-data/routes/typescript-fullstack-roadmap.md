# TypeScript 全栈项目路线

## 路线目标

用一个本地优先的 Markdown 管理工具练习 TypeScript 全栈工程能力，覆盖 API 设计、文件存储、前端状态管理、测试和发布自动化。

## 学习主线

| 模块 | 关键能力 | 推荐练习 |
| --- | --- | --- |
| 后端 | Express API、输入校验、文件读写安全 | 新增文件创建、重命名、归档接口 |
| 前端 | React 状态管理、编辑器集成、响应式布局 | 编辑器分屏、文件树、搜索过滤 |
| 数据 | Markdown 约定、IndexedDB、本地缓存 | 草稿版本、标签收藏、偏好导入导出 |
| 工程 | Vitest、CI、构建拆分、发布文档 | typecheck/test/build 自动化 |

## 8 周节奏

| 周次 | 重点 | 验收方式 |
| --- | --- | --- |
| 1 | 跑通项目和数据目录 | 本地能启动前后端 |
| 2 | 理解后端文件模型 | 能新增一个安全的 API 测试 |
| 3 | 接入编辑器核心 | 能编辑并保存 Markdown |
| 4 | 做内容管理体验 | 能搜索、过滤、置顶文件 |
| 5 | 增强本地可靠性 | 刷新页面后草稿不丢失 |
| 6 | 优化预览和性能 | 大文档滚动和切换不卡顿 |
| 7 | 补齐测试和 CI | PR 自动完成检查 |
| 8 | 发布准备 | README、贡献指南、变更日志齐全 |

## 风险清单

- Markdown 文件路径必须限制在数据目录内。
- 自动保存不能覆盖用户未确认的服务端内容。
- IndexedDB 数据结构升级要兼容旧版本。
- 前端拆分后要确认懒加载失败时有可见反馈。
